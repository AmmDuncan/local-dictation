User DECISION: doesn't care about the warning (friends know him) ‚Üí notarization NOT pursued; scaffold stays.
- **MODE COLLAPSE (user-driven)**: removed the whole multi-mode system. User insight: only Corrector ever changed words, so Clean/Email/Code silently failed to fix names ("clot"‚Üí"Claude"), and the picker had no descriptions = confusing. A "mode" was just a different system-prompt to Qwen (+ whether vocab was injected ‚Äî the bug: only Corrector got vocab). NOW: **one polish behavior = always correct mishearings (vocab+defaults context) + cleanup**, guarded by `isFaithfulCorrection`. DELETED: DictationMode, AppProfile/Resolver, AppsTab/ProfilesModel, Mode pickers (menu+Settings), mode/profile settings keys. TranscriptPolisher holds the single corrector prompt (hardened: "keep every other word, only substitute"); LlamaPolishEngine/ServerBackedPolisher always pass vocab context. Corrector prompt VERIFIED via bundled Qwen: "clot"/"cloud"‚Üí"Claude" fixed, "cloud cover"/"blood clot" kept.
- **KEY LEARNINGS**: (1) whisper --prompt biasing is PARTIAL for stubborn homophones (Claude‚Üícloud/clot) ‚Äî the LLM correction is what reliably fixes them, and it must (a) be allowed to change words + (b) be fed the vocab. (2) vocab is the lever, not model recency (no-vocab: neither Qwen nor Gemma fixes names). (3) Gemma 3 4B A/B: Qwen 6/6 vs Gemma 4/6 (Gemma over-corrects genuine cloud); both ~0.2s. Qwen stays default. Polish-model selector (Models tab, Qwen+Gemma) shipped earlier in v0.2.x.
